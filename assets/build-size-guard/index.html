Why teams use it

Build size growth is usually silent until it is expensive.

A bad texture import, a larger mesh, an Addressables shift, or a compression change can land in source control without anyone noticing.

  • CI times spike because the build suddenly got heavier.
  • Store uploads fail later than they should.
  • Players feel larger downloads and patches.
  • Release reviews become reactive instead of predictable.

Build Size Guard makes size changes visible early enough to act on them.

Diff the regression

See exactly what changed.

Get ranked signal instead of noise when a build grows.

  • Total size delta and percentage increase.
  • Top contributors to the growth.
  • Category-level increases.
  • Newly added and removed content.

Enforce the limit

Turn size budgets into release rules.

Define the boundaries that matter to your shipping process.

  • Total build size budget.
  • Artifact size on disk.
  • Percentage growth limit.
  • Absolute delta cap.
  • Per-category budgets for textures, audio, meshes, shaders, and code.

When a limit is exceeded, CI can fail with deterministic exit codes and exported reports.

CI Ready

Run it in batchmode and treat size like a real metric.

  • Deterministic output for automation.
  • JSON export for pipelines.
  • Markdown export for readable reports.
  • Automated baseline comparison during CI runs.

Track history

Keep a clean record of build snapshots over time.

  • Create a snapshot on every build.
  • Compare to the latest passing baseline.
  • Store build history for review.
  • Track regression trends across releases.

Who it is for

Built for teams that need release guardrails.

  • Unity teams shipping to mobile.
  • Developers using CI/CD.
  • Studios tired of size surprises.
  • Projects using Addressables.
  • Teams shipping patches and updates.

Who it is not for

Not meant for teams that do not manage build size.

  • Projects that do not track builds.
  • Teams that do not care about install size.
  • Runtime optimization workflows, since this is editor-only.

FAQ

Common questions before rollout.

Does this modify my build?

No. It only reads BuildReport data and file sizes.

Does it ship in the final build?

No. It is editor-only.

Does it support CI?

Yes. It includes a batchmode runner with stable exit codes.

Can I export reports?

Yes. JSON and Markdown are supported.
