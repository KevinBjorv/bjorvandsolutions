IL2CPP Strip Guard documentation

IL2CPP Strip Guard is a Unity Editor utility that scans a Unity project for common IL2CPP managed stripping, reflection, serialized type reference, and AOT generic risk patterns before they become build-only bugs.

Core promise: Find IL2CPP stripping risks before your build breaks. The tool provides a practical scan and reviewable fixes; it does not prove that a project is perfectly IL2CPP-safe.
  • Documentation source date: May 5, 2026
  • Publisher: Kevin bjorvand / Bjorvand Solutions
  • Support: kevin@smartindie.dev or Bjorvand Solutions support
  • Product page: IL2CPP Strip Guard homepage

1. Requirements and target users

Minimum supported Unity target:

Unity 2021.3 LTS+

Preferred tested versions:

  • Unity 2021.3 LTS
  • Unity 2022.3 LTS
  • Unity 6+

The current validation workspace was created with Unity 6000.4.4f1.

The tool is for Unity developers shipping to IL2CPP targets such as WebGL, Android, iOS, VR, console-like platforms, and any project using IL2CPP as the scripting backend.

2. Release package scope

Release format:

Unity Asset Store .unitypackage export

Include only the releasable product folder:

Assets/IL2CPPStripGuard/**

Exclude generated output:

  • Assets/IL2CPPStripGuard/Generated/**
  • Assets/IL2CPPStripGuard/Generated.meta

Exclude validation project content such as Assets/IL2CPPStripGuardTests/**, scenes, project settings, package settings, Library, Logs, Temp, and UserSettings. The sample under Assets/IL2CPPStripGuard/Samples is part of the product asset folder and is included.

3. Installation

  1. Import the asset into a Unity project.
  2. Let Unity compile the editor assembly.
  3. Open the tool from Tools > IL2CPP Strip Guard.

Product code is editor-only and is contained in:

Assets/IL2CPPStripGuard/Editor

Assemblies:

  • KevinBjorvand.IL2CPPStripGuard.Editor
  • KevinBjorvand.IL2CPPStripGuard.Samples

Editor tests live outside the release package under Assets/IL2CPPStripGuardTests. The asset should import cleanly without external paid dependencies.

4. Quick start

  1. Import the asset.
  2. Open Tools > IL2CPP Strip Guard.
  3. Click Scan IL2CPP Risks.
  4. Review the Dashboard and Findings tabs.
  5. Open Generated Fixes.
  6. Click Preview link.xml to inspect suggested preserve entries.
  7. Click Generate link.xml when you want to write the generated file.
  8. Click Export Report to write JSON and Markdown reports.
  9. Configure scan options, output paths, thresholds, and ignore rules in Project Settings > IL2CPP Strip Guard.

The tool never modifies user source code, player settings, existing user link.xml files, or build settings automatically.

5. Main Unity entry points

  • Editor window: Tools > IL2CPP Strip Guard
  • Project settings: Project Settings > IL2CPP Strip Guard
  • Settings asset: Assets/IL2CPPStripGuard/Settings/IL2CPPStripGuardSettings.asset
  • CI method: IL2CPPStripGuard.Editor.IL2CPPStripGuardCI.Run
  • Fully namespaced CI method: KevinBjorvand.IL2CPPStripGuard.Editor.IL2CPPStripGuardCI.Run

6. User interface

Dashboard tab

  • Scan IL2CPP Risks
  • Generate link.xml
  • Export Report
  • Open Settings
  • Last scan date, Unity version, scripting backend, build target, severity/status counts, and CI threshold result

Findings tab

  • Filters: All, Critical, High, Medium, Low, Covered, Ignored

Generated Fixes tab

  • Optional Generate AOT Hints button when enabled in settings

7. Finding severity and status

SeverityMeaning
CriticalString-based dynamic creation, especially assembly/type string overloads such as Activator.CreateInstance("Assembly", "Type").
HighString-based type lookup, dynamic type construction, open generic type metadata, MakeGenericType, MakeGenericMethod, and likely IL2CPP AOT risks.
MediumReflection member lookup, reflected invocation, managed reference type names in serialized data, and namespaced serialized strings that look like type names.
LowLower-confidence signals such as SerializeReference usage or simple type-like registry strings.
StatusMeaning
ActiveThe risk is currently uncovered and not ignored.
CoveredThe risk appears covered by a recognized preserve source.
IgnoredThe risk matches a user ignore rule or source comment and remains visible in the Ignored filter.

8. Scan inputs

The scanner inspects C# files, serialized files, existing Assets link.xml files, generated preserve files, and Preserve attributes in scanned C# files.

Serialized file extensions scanned:

  • .asset, .prefab, .unity, .controller, .overrideController
  • .playable, .inputactions, .json

Package C# and serialized files are scanned only when Scan Packages is enabled.

Common exclusions include .meta files, Library, Temp, Obj, generated folders when disabled, and AddressableAssetsData when Scan Addressables is disabled.

9. C# risk detection

The C# scanner detects common risky patterns including:

  • Type.GetType("Namespace.Type") and assembly.GetType("Namespace.Type")
  • Activator.CreateInstance("Assembly", "Type"), Activator.CreateInstance(type), and Activator.CreateInstance(typeof(SomeType))
  • GetMethod, GetField, GetProperty, MethodInfo.Invoke, FieldInfo.GetValue, and FieldInfo.SetValue
  • MakeGenericType, MakeGenericMethod, typeof(Generic<>), and [SerializeReference]

The scanner also tracks simple local variables where possible, including Type variables assigned from typeof(SomeType) and MethodInfo or FieldInfo variables associated with typeof(SomeType).

10. Serialized asset risk detection

The serialized scanner detects:

  • managedReferenceFullTypename
  • managedReferenceFieldTypename
  • Serialized strings that look like namespaced type names
  • Serialized key/value pairs where the key suggests type, class, command, action, event, factory, handler, or serializer usage and the value looks like a type name

The scanner intentionally skips common false-positive shapes such as YAML directive lines, very long serialized lines, URLs, file paths, numeric values, hex-like identifiers, and values beginning with JSON/YAML anchors or collection markers.

11. Preserve coverage detection

Recognized coverage sources:

  • [UnityEngine.Scripting.Preserve], [Preserve], and [assembly: Preserve]
  • Existing link.xml files under Assets
  • Generated link.xml under Assets/IL2CPPStripGuard/Generated
  • Assembly-level, namespace-level, type-level, and applicable member-level preserve entries

Only link.xml files under Assets are counted as Unity-applied preserve coverage. Package-local link.xml files are not counted as coverage.

Nested type names are normalized internally with +. Generated link.xml output uses Unity's nested type separator /.

12. Generated link.xml

Default output path:

Assets/IL2CPPStripGuard/Generated/link.xml

The generator writes valid XML, groups entries by assembly, adds explanatory comments, avoids duplicate assembly/type entries, and uses preserve="all" for generated type entries.

Safety behavior:

  • Includes active findings with a concrete type name.
  • Keeps generated-covered entries during regeneration.
  • Does not include AOT-only findings with no concrete type.
  • Does not overwrite existing user link.xml files.
  • Refuses generated output paths outside Assets/IL2CPPStripGuard/Generated.
  • Refuses to replace an existing file that does not look generated by IL2CPP Strip Guard.

Generated link.xml files are suggestions and are safe to delete.

13. Generated AOT hints

Default output path:

Assets/IL2CPPStripGuard/Generated/IL2CPPAotHints.cs

AOT hint generation is disabled by default. When enabled, the current generator writes a safe placeholder class unless the scanner can infer concrete generic references without guessing.

link.xml can preserve metadata, but it may not force every closed generic instantiation. Add explicit runtime references or hand-authored AOT hints when a project depends on dynamic generic construction.

14. Report export

Default report folder:

Library/IL2CPPStripGuard/Reports

Export Report writes JSON and Markdown reports with scan date, Unity version, scripting backend, build target, CI result, total findings, severity counts, covered and ignored counts, grouped findings, explanations, suggested fixes, coverage source, ignore reason, and generated link.xml preview.

Report output paths must stay under Library/IL2CPPStripGuard/Reports.

16. CI mode

Unity command line method:

-executeMethod IL2CPPStripGuard.Editor.IL2CPPStripGuardCI.Run

Fully namespaced equivalent:

-executeMethod KevinBjorvand.IL2CPPStripGuard.Editor.IL2CPPStripGuardCI.Run

CI mode loads saved settings, runs the scanner without opening the window, builds a generated link.xml preview, exports JSON and Markdown reports, logs severity and status counts, uses configured fail thresholds, exits with code 1 when thresholds are exceeded, and exits with code 0 when thresholds are not exceeded.

Default CI thresholds fail on active Critical and High findings.

19. Safety, limitations, troubleshooting, and release checklist

Safety guarantees

The tool must not modify user source code automatically, overwrite existing user link.xml files automatically, delete user files, force player builds, or change project settings without explicit user action.

Allowed writes require a clear user action: save settings, save ignore rules, generate link.xml, generate AOT hints, or export reports.

Limitations

  • The scanner is heuristic.
  • It does not run player builds or upload/validate platform builds.
  • It does not perform deep third-party package-specific analysis.
  • It does not prove generic AOT coverage.
  • Generated link.xml preserves metadata but may not force every generic method body or closed generic instantiation.
  • False positives and false negatives are possible.

Troubleshooting

  • No settings asset in CI: open the Unity project once or create/save settings through Project Settings > IL2CPP Strip Guard.
  • No findings appear: confirm scan scope for packages, generated folders, and Addressables.
  • A covered type still appears: covered findings remain visible for auditability.
  • A package link.xml is not counted: only link.xml files under Assets are counted as project coverage.
  • Generated link.xml refuses to write: keep the path under Assets/IL2CPPStripGuard/Generated, ending in .xml, and avoid replacing non-generated files.
  • CI fails: check configured fail thresholds. By default, active Critical and High findings fail CI.

Release checklist

  • Import the asset into a clean Unity project and confirm no compile errors.
  • Run Edit Mode tests.
  • Open Tools > IL2CPP Strip Guard, scan, preview link.xml, and export reports.
  • Run the demo sample and confirm expected demo findings.
  • Confirm release export includes only Assets/IL2CPPStripGuard/** and excludes generated output.
  • Confirm README, sample README, release scope, and project documentation are up to date.
