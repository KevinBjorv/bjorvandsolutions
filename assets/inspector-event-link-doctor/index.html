Unity Editor Tool

Inspector Event Link Doctor for Unity

Find silent Inspector event breakage before your game does.

Scan serialized Unity callback data and see where each Inspector-connected event link lives, what target and method it depends on, whether it is valid, broken, risky, ignored, or unresolved, and what to review next.

  • UnityEvent validation
  • Animation Event checks
  • Prefab and ScriptableObject scans
  • Export reports
  • Batchmode CI validation

Why teams use it

Inspector callback links can break silently after normal Unity refactors.

Unity projects often connect logic through Button clicks, UI value changes, custom UnityEvents, ScriptableObject events, prefab references, and Animation Events. Those links are serialized as method names and object references, so a rename, removed component, wrong signature, or missing receiver can stay hidden until runtime.

  • Buttons can point to deleted methods.
  • Prefabs can keep dead persistent event calls.
  • Animation Events can call methods that no longer exist.
  • ScriptableObjects can hold callback references that need review.

Scan Scope

Scan current scenes, build scenes, selections, or the entire project.

The scanner covers supported serialized callback sources under Assets and skips noisy project folders by default.

  • Open active scene and enabled Build Settings scenes.
  • Selected assets or selected folders from the Project window.
  • Prefab assets, ScriptableObject assets, and AnimationClip assets.
  • Optional all-Assets scene scanning through settings.

UnityEvents

Generic persistent-call validation.

The scanner walks serialized properties looking for Unity persistent-call data, so it can cover UI events, custom UnityEvents, typed UnityEvents, nested serializable fields, prefab assets, scene objects, and ScriptableObjects.

  • Detects missing targets, empty methods, missing methods, wrong signatures, unresolved types, and invalid call modes.
  • Flags risky private, obsolete, disabled, inactive, ambiguous, editor-only, prefab-instance, and cross-prefab patterns.

Animation Events

Clip checks with receiver context when available.

AnimationClip events are reported with clip, event index, timing, function name, parameters, source asset, and receiver context when the scan can find one.

  • Marks empty names and known missing receiver methods as broken.
  • Separates context-dependent clips into Unresolved Context when a receiver is not known.

Workflow

Review, locate, ignore known-safe noise, and rescan.

The main window exposes status chips, filters, paging, row details, and practical row actions so each result can be traced back to the exact source.

  • Use Ping Source, Select Source, Open Scene, Ping Target, Copy Method, Copy Path, and Copy Issue.
  • Ignore or unignore specific results, folders, and issue types through Project Settings.
  • Export Markdown, JSON, TXT, HTML, CSV, or XML reports for review and handoff.

CI

Fail batchmode validation when blocking callback issues exceed thresholds.

Run KevinBjorvand.EventLinkDoctor.EventLinkDoctorCI.RunValidation from Unity batch mode, optionally export JSON, and configure whether broken and risky issue counts should fail the run.

Who it is for

Built for Unity projects with serialized callback links.

  • Solo and small-team Unity developers.
  • UI-heavy Unity games and tools.
  • Projects that use prefabs and ScriptableObjects heavily.
  • Teams that want callback validation in CI.

What it is not

Focused reporting, not automatic repair.

  • Not a graph editor.
  • Not a repair engine.
  • Not a runtime debugger.

FAQ

Common evaluation questions.

Does this repair broken callbacks?

No. Version 1.0.0 reports the source and suggested fix. Automatic repair is intentionally out of scope.

Does this require runtime code?

No. The product scanner, window, exporter, settings, and CI command are editor-only. Demo scripts support the included sample scenes and are not auto-referenced by customer assemblies.

Does it require TextMeshPro, Timeline, or Addressables?

No. The scanner uses generic serialized UnityEvent discovery. TextMeshPro events are detected when TextMeshPro is installed because they serialize through the same persistent-call structure.

Can it run in CI?

Yes. It includes a Unity batchmode entry point that exports JSON when configured and fails according to broken or risky issue thresholds.
