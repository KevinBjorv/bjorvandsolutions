Why teams use it

Shader variant growth is easy to miss in raw logs.

This tool turns variant counts into snapshots, diffs, and budget checks your team can review.

  • Capture snapshots after builds and track counts over time.
  • Compare a known baseline against the current result instead of scanning raw logs by hand.
  • Fail CI at the point where the regression is introduced.

Compare builds

See exactly what changed between baseline and current.

  • Summary cards for baseline total, current total, delta, and percent delta.
  • Filtered diff views with search and sorting.
  • Explicit baseline/current selection or direct path-based compare.

Find the cause

Catch top offenders instead of treating the whole diff like noise.

  • Search and filter shader names in the diff results.
  • Sort by delta, total variants, or percent delta.
  • Review increases, decreases, new entries, and removed entries separately.

Budget guardrails

Turn variant limits into clear production rules.

  • Hard and warning limits for total variants, delta variants, and percent delta.
  • Per-shader caps for shaders that must stay under a known ceiling.
  • Ignore rules and allow rules so reports stay focused on what matters to your team.

Clear outputs

Get readable results when a snapshot breaks the rules.

  • Clear warnings and blocking violations when budgets are crossed.
  • Diagnostics inside the editor window for parser health and data confidence.
  • Incomplete snapshot gating so bad data does not quietly pass.

CI ready

Gate builds automatically with reports, baselines, and stable exit codes.

  • Compare against latest, latestPassing, or a specific snapshot path.
  • Use profile-aware baseline matching and latest-passing index overrides.
  • Write JSON and Markdown reports directly into CI artifacts.

Who it is for

Built for teams that want structure around shader variant growth.

  • Multi-target projects where variant creep turns into real build pain.
  • Teams reviewing build health in CI or on shared branches.
  • Projects that want repeatable baselines instead of memory and guesswork.

Who it is not for

Not meant for runtime shader optimization or teams that never inspect build outputs.

  • Runtime GPU profiling workflows, since this tool operates on editor and batchmode build data.
  • Projects that do not need baselines, budgets, or regression checks.
  • Teams expecting a runtime package or in-game UI.

FAQ

Common questions before rollout.

Does this change shaders or build output?

No. It reads variant information from Editor.log, stores snapshots, compares them, and evaluates budgets.

Can it fail CI automatically?

Yes. It includes a batchmode runner with stable exit codes, external budget JSON support, and report export options.

Can I compare explicit snapshot files?

Yes. The editor window supports path-based comparison, and batchmode supports explicit baseline and current paths.

What happens if a snapshot is incomplete?

Incomplete snapshots are hidden by default in the editor and fail the batch check by default so bad data does not quietly pass.
