Enforce Unity import settings automatically, validate drift, and fix issues fast.

Import Settings Validator & Fix uses deterministic rules to keep texture, audio, and model import settings consistent across teams, machines, and asset drops.

  • Fix selected or fix all plus exportable compliance reports

The problem

Unity projects drift over time. Import settings vary by person, machine, source package, and timeline. That inconsistency can bloat builds, reduce visual or audio quality, and create repetitive manual importer editing work.

What it does

  1. Match assets using ordered rules by path pattern and importer type
  2. Auto-apply expected settings during import through Unity import hooks
  3. Scan the full project and flag non-compliant assets with readable diffs
  4. Fix selected items or fix all in scope, then export compliance reports
MVP scope is focused by design: import enforcement, validator, fix workflow, and report export.

Core MVP capabilities

Rule system

ImportRuleSet ScriptableObject with ordered rules. Last match wins for predictable outcomes. Rules can target TextureImporter, AudioImporter, and ModelImporter.

Auto enforcement on import

Uses pre-process hooks for texture, audio, and model importers. Applies only when settings differ to avoid churn and noise.

Validator and fix flow

Scan Assets/ to compute expected settings, compare against current importer state, then fix selected rows or all filtered rows.

Reporting

Export compliance summaries with totals and per-asset rows, including path, importer type, status, applied rule, and diff summary.

Validator and fix workflow

Validator output

  • Compliant / non-compliant / skipped status
  • Asset path and importer type
  • Rule applied
  • Human-readable diff summary

Safety controls

  • Global enforcement on/off toggle
  • Dry-run mode for previewing changes
  • Fix All confirmation dialog with counts
  • Concise logging with configurable verbosity

Planned MVP stages

  1. Foundations: rules, matching engine, expected settings computation, explain output
  2. Enforcement: import hooks, preset apply, change detection, project toggles
  3. Validator core: full scan, compliance evaluation, readable diffs
  4. Fix actions: fix selected, fix all, dry-run preview, reimport handling
  5. Reporting and polish: CSV/JSON export, UX filtering, docs and sample RuleSet

MVP non-goals

  • No visual rule builder UI in MVP
  • No heuristic optimization or auto-suggested settings
  • No build pipeline or CI hook integration in MVP
  • No runtime components. Editor-only tool.
Scope discipline keeps behavior deterministic and implementation practical for teams that need reliable enforcement first.

Definition of done focus

  • Deterministic behavior across machines and reimports
  • No infinite reimport loops or continuous dirty state
  • Clear explainability for why a rule changed an importer
  • Example RuleSet and Presets included for quick setup

FAQ

Q: Does this tool choose import settings for me automatically?

A: No. You define standards through rules and presets. The tool enforces what you declared.

Q: Can I preview before applying changes?

A: Yes. Dry-run modes are part of the MVP scope so you can review intended changes before mass updates.

Q: Which importer types are covered in MVP?

A: TextureImporter, AudioImporter, and ModelImporter.

Q: Will this add runtime code to my game build?

A: No. The tool is editor-only.

Q: Can it export compliance data for reviews?

A: Yes. CSV export is required in MVP, with JSON export included when practical.
