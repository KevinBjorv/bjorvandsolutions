Unity Serialization Migration Guard technical documentation

Unity Serialization Migration Guard is an editor-only migration workflow for serialized field renames. It helps teams audit rename mappings, scan text-serialized assets for legacy tokens, preview scoped reserialize work, and prove when cleanup is safe.

Coverage Scan and Cleanup Proof require Project Settings > Editor > Asset Serialization Mode = Force Text. Audit can run without it.

1. What you get

  • Audit warnings for missing types, ambiguous type names, missing target fields, non-serializable target fields, and missing FormerlySerializedAs.
  • Coverage reporting with counts per field and per asset, plus unreadable asset tracking.
  • Reserialize preview, scoped execution, progress reporting, cancellation, and optional backups.
  • Cleanup proof that tells you when it is safe to remove legacy mappings and old paths.
  • JSON and CSV exports for audit, coverage, and cleanup reports.
  • Batchmode commands and exit codes for CI gates.

2. Access paths

  • Window: Tools/Unity Serialization Migration Guard/Open Window
  • Project Settings: Project/Unity Serialization Migration Guard
  • Project Settings shortcut: Tools/Unity Serialization Migration Guard/Open Project Settings
  • Batch entrypoint: RefactorMigrationSafetySuite.Batch.RMSSBatchEntryPoint.Run

Window title: RMSS. Minimum size: 760 x 420. Tab order: Audit, Coverage Scan, Reserialize, Cleanup, Settings.

3. Supported asset scope

  • Scenes: .unity
  • Prefabs: .prefab
  • ScriptableObject assets: .asset, including parent assets with embedded ScriptableObjects

Coverage is token-based inside text-serialized YAML. Unsupported asset types are skipped and shown as skipped inputs.

4. Audit tab

Audit validates rename mappings against the current project code before you trust them in a migration run.

  • Git Diff Import supports pasted diff text or a diff file path, plus preview, append, and replace actions.
  • Manual Rename Mappings supports add, clear, JSON import/export, and row removal.
  • Checks include missing types, ambiguous names, missing target fields, non-serializable target fields, missing FormerlySerializedAs, and lower-risk mapping issues.

5. Coverage scan

  • Requires valid mappings with non-empty Type, Old, and New values.
  • Requires Force Text serialization.
  • Reports candidate assets, scanned assets, unreadable assets, affected assets, total legacy token occurrences, and supported-scope completeness.
  • Orders per-field summary by total occurrences, affected assets, type name, then legacy field.

6. Reserialize workflow

  • Coverage Results: use current coverage hits.
  • Folder: recurse under a selected folder.
  • Selection: use the current Project window selection and expand folders.
  • Preview shows source, asset count, skipped inputs, and warnings before the run starts.
  • Scenes use scene save flow, prefabs use prefab save flow, and other supported assets use AssetDatabase.ForceReserializeAssets.

7. Cleanup proof

  • IsSafeToRemoveLegacyMappings
  • IsSupportedScopeComplete
  • RemainingLegacyTokenOccurrences
  • ScopeSummary
  • Notes[]

Cleanup is unsafe when there is no coverage report, when legacy tokens still remain, or when supported scope is incomplete because assets could not be read.

8. Settings and storage

  • Enable Verbose Logging
  • Create Backups By Default
  • Default Export Directory
  • Backup Directory

Settings are stored in ProjectSettings/RefactorMigrationSafetySuiteSettings.asset. Relative paths resolve from the Unity project root. Backup paths inside Assets or Packages are rejected.

9. Mapping JSON

{
  "schemaVersion": 1,
  "mappings": [
    {
      "typeName": "MyCompany.CharacterState",
      "oldName": "oldHealth",
      "newName": "currentHealth"
    }
  ]
}

Fields: schemaVersion, mappings[], typeName, oldName, and newName.

10. Exports and reports

  • Audit: JSON and CSV with input mappings, risks, and counts
  • Coverage: JSON and CSV with metrics, unreadable assets, per-field summary, and per-asset hits
  • Cleanup: JSON and CSV with safe state, completeness, remaining tokens, and notes

11. Batchmode and CI

Run Unity with -executeMethod RefactorMigrationSafetySuite.Batch.RMSSBatchEntryPoint.Run.

OptionMeaning
--mappingsJson <path>Required for coverage and cleanup. Optional for audit.
--jsonOut <path>Optional JSON output path.
--failOnRisk true|falseFail audit when high-risk findings exist.
--failOnCoverage true|falseFail coverage or cleanup when remaining risk exists.
Exit codeMeaning
0Success
1Parse, input, or setup error
2Audit failure with high risk and fail-on-risk enabled
3Coverage or cleanup failure with fail-on-coverage enabled

12. Practical workflow

  1. Build mappings manually or import candidates from a unified git diff.
  2. Export mappings to JSON for reuse and traceability.
  3. Run Audit and clear high or medium risks before changing asset scope.
  4. Enable Force Text serialization if you need coverage or cleanup.
  5. Run Coverage Scan and review unreadable assets.
  6. Build a Reserialize preview and optionally enable backups.
  7. Run Reserialize for the specific scope that needs migration.
  8. Run Cleanup Proof before removing legacy attributes or old mapping paths.

13. Requirements and notes

  • Coverage Scan and Cleanup Proof require Force Text serialization.
  • Coverage is token-based. If multiple fields share the same token name, results can be ambiguous.
  • Unsupported asset types are skipped and shown as skipped inputs.
  • Selection scope only considers Project window assets, not arbitrary scene selections.
