Why teams use it

Import settings drift over time unless someone keeps policing it.

Textures, audio, models, and sprites all pick up inconsistent settings when multiple contributors are touching the project.

  • Wrong compression settings push build size higher.
  • Incorrect load types create runtime inconsistency.
  • Bad model scales and mismatched sprite settings slow review.
  • Manual inspection does not scale as the asset count grows.

The result is a cleaner way to enforce standards across the project, not a last-minute cleanup pass.

FAQ

Common questions before enabling enforcement.

Does it automatically change assets?

Only when enforcement or fix mode is enabled. Otherwise, it reports violations so you can review them first.

Can it validate audio and model imports too?

Yes. The tool supports texture, audio, and model import checks, so you can enforce more than texture compression rules.

Does this run automatically?

Yes. Enforcement can be enabled when you are ready to apply the configured rules automatically.
