Inspector Event Link Doctor documentation

Inspector Event Link Doctor is a Unity Editor utility for finding broken and risky Inspector-connected callbacks before runtime.

The tool is a focused report utility for serialized Inspector callback links. It is not a graph editor, repair engine, or runtime debugger.

1. Requirements

  • Source project version: 1.0.0.
  • Current source project Unity version: 6000.4.4f1.
  • Target support currently claimed in the source documentation: Unity 6.
  • The editor assembly is Editor-only and does not add required runtime code to user projects.

2. Quick start

  1. Open Tools > Inspector Event Link Doctor > Open Window.
  2. Click Current Scene.
  3. Review Broken, Risky, and Unresolved Context results.
  4. Select a result row to inspect full details.
  5. Use Ping Source, Select Source, Open Scene, Ping Target, Copy Method, Copy Path, Copy Issue, or Ignore as needed.
  6. Fix the serialized callback in the Inspector.
  7. Re-scan.
  8. Optionally export a report or run CI validation.

For a project-wide pass, click Entire Project. For demo validation, click Demo.

3. Scan modes

ModeScope
Current SceneScans the active loaded scene.
Build ScenesScans enabled scenes from Unity Build Settings.
SelectedScans selected assets or folders from the Project window.
Entire ProjectScans supported assets under Assets, with optional all-scene inclusion through settings.
DemoScans the included demo folder with package demo inclusion enabled.

4. What it scans

Supported source asset types include open active scenes, enabled Build Settings scenes, selected or all-Assets scene assets, prefab assets, ScriptableObject assets, and AnimationClip assets.

  • Serialized UnityEvent persistent calls.
  • UI events that serialize through UnityEvent, including Button and Toggle.
  • Custom UnityEvent fields on MonoBehaviours.
  • Custom typed UnityEvents for common primitive and Unity object argument types.
  • UnityEvents inside nested serializable fields when visible through Unity serialization.
  • UnityEvents on ScriptableObjects.
  • Animation Events on AnimationClip assets.
  • Missing scripts on GameObjects and scanner exceptions as reportable errors.

By default, scans skip Library, Temp, Logs, Packages, generated folders, user-configured ignored folders, package documentation, and package demo content.

5. UnityEvent validation

The scanner walks SerializedObject visible properties and looks for the serialized persistent-call structure m_PersistentCalls.m_Calls.

  • Broken results include missing or null targets, empty method names, unresolved target types, invalid listener modes, missing methods, and wrong signatures.
  • Risky results include disabled calls, ambiguous compatible methods, private methods, obsolete methods, editor-only target types, disabled components, inactive GameObjects, missing target scripts, prefab instance sources, and cross-prefab references.
  • Valid results have an existing target, matching method signature, and no detected risk flags.

Return type must be void. Parameter count and type must match the expected persistent listener mode.

6. Animation Events

The scanner reads Animation Events from AnimationClip assets through Unity editor APIs and records clip, event index, time, function name, parameters, source path, and receiver context when known.

  • Without receiver context, empty function names are Broken and non-empty functions are Unresolved Context.
  • With receiver context, missing receiver methods and incompatible signatures are Broken.
  • Object parameter gaps, multiple compatible methods, duplicate same-time events, and suspicious function names are Risky.

Animation Events are context-dependent. A clip can be valid in one receiver context and broken or ambiguous in another.

7. Result statuses

StatusMeaning
ValidThe callback resolves cleanly.
RiskyThe callback currently resolves, but a fragile or suspicious condition should be reviewed.
BrokenA required target, type, method, signature, or serialized mode is invalid.
IgnoredThe result matched an ignored folder, ignored issue type, or specific result ignore rule.
Unresolved ContextThe scanner needs more receiver information before it can classify an Animation Event.

8. Main window

Open the window from:

Tools > Inspector Event Link Doctor > Open Window
  • Docs button, scan toolbar, export format selector, export button, and Project Settings button.
  • Status summary chips and status, source, and kind filters.
  • Paged result list and result details panel.
  • Default visible filters: Broken, Risky, and Unresolved Context.
  • Default hidden filters: Valid and Ignored.

9. Settings and ignores

Open settings from:

Project Settings > Inspector Event Link Doctor
  • Configure scan defaults, result paging, export defaults, and Advanced CI settings.
  • Add, reveal, and remove ignored project folders under Assets.
  • Ignore issue types globally when a category is accepted noise.
  • Ignore a specific result by source GUID, source object global ID, event field path, target global ID, method name, and issue type.

Ignored results do not fail CI by default. If CI includes ignored results, each ignored row is counted by its original status for threshold checks.

10. Exports

Supported report formats:

  • Markdown (.md)
  • JSON (.json)
  • Text (.txt)
  • HTML (.html)
  • CSV (.csv)
  • XML (.xml)

Common report metadata includes tool version, scan date UTC, Unity version, project name, project path, summary counts, and result list. JSON is intended for CI and external tooling.

11. CI validation

Run the static batchmode entry point:

Unity -batchmode -quit -projectPath "ProjectPath" -executeMethod KevinBjorvand.EventLinkDoctor.EventLinkDoctorCI.RunValidation

Useful arguments:

-eventLinkDoctorOutput "Reports/EventLinkDoctorReport.json"
-eventLinkDoctorBrokenThreshold 0
-eventLinkDoctorFailOnRisky true
-eventLinkDoctorRiskyThreshold 0
-eventLinkDoctorIncludeIgnored false
-eventLinkDoctorIncludeAllScenes true
-eventLinkDoctorIncludePackageDemo false
  • Uses saved settings unless command-line overrides are supplied.
  • Exports JSON when an output path is configured.
  • Exits with code 1 if broken results exceed the broken threshold.
  • Optionally exits with code 1 if risky results exceed the risky threshold.
  • Exits with code 0 when thresholds pass.

12. Demo, limitations, and support

Demo content is located under:

Assets/InspectorEventLinkDoctor/Demo
  • Demo scenes intentionally include valid, broken, risky, and unresolved examples.
  • Demo content is ignored by normal scans by default.
  • Rebuild demo content from Tools > Inspector Event Link Doctor > Rebuild Demo Content.
  • License source status: Inspector Event Link Doctor License
  • Support page: Bjorvand Solutions Support

13. FAQ

Does this repair broken callbacks?

No. Version 1.0.0 reports the exact source and suggested fix. Automatic repair is intentionally out of scope for the MVP.

Does this require runtime code?

No. Product scanner, window, exporter, settings, and CI command code is editor-only.

Why is my AnimationClip marked Unresolved Context?

The clip has Animation Events, but the scanner did not find a scanned scene or prefab where the clip is assigned to a receiver.
