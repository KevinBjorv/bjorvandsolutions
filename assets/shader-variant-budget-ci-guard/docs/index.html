Shader Variant Budget & CI Guard technical documentation

Shader Variant Budget & CI Guard is an editor-only Unity tool for capturing shader variant snapshots from Editor.log, comparing a baseline against a current build, applying budgets, and gating CI with stable exit codes.

Incomplete snapshots are hidden by default in the editor and, by default, fail batchmode so broken capture data does not quietly hide regressions.

1. What it does not do

  • No runtime or in-game UI.
  • No editor-window export for full compare reports. Full JSON and Markdown reports are batchmode features.
  • No target-specific budget policy editor in the Project Settings UI. The UI edits the default policy block.
  • No file picker for path-based compare fields. Snapshot paths are entered manually.

2. Access paths

  • Window: Tools/Shader Variant Guard/Open Window
  • Project Settings: Project/Shader Variant Guard
  • Batch entrypoint: ShaderVariantGuard.Editor.Cli.ShaderVariantGuardBatchRunner.Run

3. Storage layout

  • ShaderVariantGuardData/snapshots
  • ShaderVariantGuardData/reports
  • ShaderVariantGuardData/diagnostics
  • ShaderVariantGuardData/latest_passing_snapshots.json
YYYYMMDDTHHmmssfffZ_<target>_<label>_<snapshotId>.json

4. Core workflow

  1. Capture a snapshot from Editor.log.
  2. Store the snapshot as deterministic JSON.
  3. Compare a baseline snapshot against a current snapshot.
  4. Apply ignore rules to the diff.
  5. Evaluate budgets and per-shader caps.
  6. Review results in the window or emit reports in batchmode.
  7. Optionally promote the current snapshot to latestPassing when the run passes.

5. Snapshot capture

  • Manual capture: reads the current Editor.log, uses label manual, saves immediately, and selects the snapshot as current.
  • Automatic capture: runs after builds when Auto Capture Snapshots is enabled.
  • The parser recognizes shader summary lines and a trailing total shader variants summary block.
  • Capture retries or falls back to a temporary log copy when Unity is actively writing the log.

6. Snapshot completeness

  • Missing snapshot data
  • Negative totals
  • No per-shader contributors
  • Contributor totals that do not match totalVariants
  • No recognized patterns in a scanned log

Incomplete snapshots are hidden by default in the editor and fail batchmode with exit code 9 by default.

7. Editor window

  • Snapshot browser with search, target filter, Show Incomplete, and Compare Selection.
  • Compare controls for baseline/current selection, Swap, Latest vs Previous, and path-based compare.
  • Summary cards plus diff tabs for increases, decreases, new items, and removed items.
  • Diagnostics export in txt, JSON, Markdown, XML, or CSV.

9. Budget policy

  • Hard caps for total variants, delta variants, and percent delta.
  • Warning thresholds for the same metrics.
  • Per-shader caps based on current snapshot totals.
  • Ignore rules matching shader names by exact, contains, or regex.
  • Allow rules that skip matching per-shader caps.

10. Budgets JSON

Batchmode can load an external budgets file via -shaderVariantGuard.budgets <path>.

  1. Exact buildTarget plus buildProfileId
  2. Target-only policy with empty buildProfileId
  3. defaultPolicy
{
  "schemaVersion": "1",
  "defaultPolicy": {
    "enabled": true,
    "maxDeltaVariantsEnabled": true,
    "maxDeltaVariants": 500
  }
}

11. Batchmode and CI

Run Unity with -executeMethod ShaderVariantGuard.Editor.Cli.ShaderVariantGuardBatchRunner.Run.

-shaderVariantGuard.current <path|latest>latestCurrent snapshot source.
-shaderVariantGuard.baselineMatch <targetAndProfile|targetOnly|any>targetAndProfileBaseline resolution mode.
-shaderVariantGuard.budgets <path>emptyExternal budget JSON file.

Exit codeMeaning
0Success
2Blocking violation
3Invalid arguments
4Baseline resolution error
5Snapshot load error
6Report write error

8Budget policy load or parse error

12. Report outputs

  • JSON report: serialized evaluation result with diff, violations, diagnostics, baseline source, and policy source.
  • Markdown report: summary, execution context, diagnostics, blocking violations, warnings, and top increases.
  • Diagnostics export: editor-window export in txt, JSON, Markdown, XML, or CSV.

13. Troubleshooting

  • If no snapshots appear, check that capture ran and that filters are not hiding all rows.
  • If capture succeeds but the snapshot is incomplete, inspect diagnostics and verify the correct Editor.log path and tail line count.
  • If latestPassing cannot be resolved, confirm the index exists and contains a matching target/profile, or bootstrap it intentionally.
