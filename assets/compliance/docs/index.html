Third-Party Notices & Credits: License Compliance for Unity - Technical Documentation

Third-Party Notices & Credits: License Compliance for Unity is a Unity Editor tool for detecting third-party components and generating compliance artifacts.

Primary code locations

  • Assets/GameCompliancePack/Compliance/Editor (editor logic)
  • Assets/GameCompliancePack/Compliance/Runtime (runtime model/logic)

Asset was developed and tested using Unity editor version 6000.3.5f2.

What You Get

On export, the tool writes artifacts to Settings.outputDirectory (default Compliance).

Default outputs

  • THIRD_PARTY_NOTICES.txt
  • CREDITS.md
  • compliance-manifest.json
  • compliance-overrides.json (saved when overrides/manual entries change)

Output purpose

  • THIRD_PARTY_NOTICES.txt: per-component notice blocks with license text or explicit UNKNOWN/incomplete placeholders.
  • CREDITS.md: concise attribution list for credits pages.
  • compliance-manifest.json: machine-readable manifest of component metadata and resolved license state.
  • compliance-overrides.json: persistent manual decisions, manual entries, and package import monitor rules.

Feature Summary

Scanning and Detection

  • Scans UPM dependencies from Packages/manifest.json and Packages/packages-lock.json.
  • Scans configured file-system roots and native libraries in Assets.
  • Finds nearby legal files with bounded scope traversal.

License Classification

  • Uses SPDX IDs found in text.
  • Reads package metadata from package.json.
  • Falls back to text similarity against local SPDX corpus.

Overrides and Manual Entries

  • Per-component override workflow for license and attribution fields.
  • Manual entry workflow for dependencies not auto-detected.
  • Orphan manual entry handling (keep missing, remove, map).

Build and Export

  • Exports notices, credits, and manifest with added/removed/changed diff support.
  • Build-time warning/fail policies via BuildHook.
  • Package import monitor for out-of-scope .unitypackage imports.

Installation

  1. Copy Assets/GameCompliancePack/Compliance into the target project Assets folder.
  2. Open the project and allow script compilation.
  3. Open Tools/Compliance Pack.
  4. If missing, settings are auto-created at Assets/Settings/CompliancePackSettings.asset.

Notes: The scanner/resolver pipeline is local and offline (no network calls), and the build hook activates automatically through Unity pre-build integration.

Quick Start

  1. Open Tools/Compliance Pack.
  2. Use Scan and choose Scan All, Scan UPM Only, or Scan Folders Only.
  3. Review list items using filters such as Needs attention, UNKNOWN, Missing fields, source filters, and diff filters.
  4. Select a component and complete required fields in Override.
  5. For undetected dependencies, click + Manual and fill manual entry fields.
  6. Click Export Pack.

Export behavior

  • If failBuildOnUnknown is enabled and unresolved items exist, export is blocked.
  • Otherwise export writes manifest, notices, and credits, and updates the diff tab.

Scanner and Detection Details

UPM scanner (UpmDependencyScanner)

  • Inputs: Packages/manifest.json (direct) and Packages/packages-lock.json (resolved graph).
  • Skips package names starting with com.unity..
  • Adds direct dependencies plus lock-only entries not already added.
  • Source type from lock source or inferred from version spec (local, git, registry); appends -transitive when lock depth is greater than 0.
  • Component path is Packages/<packageName>.
  • Confidence: 0.8 (manifest + lock), 0.6 (manifest only), 0.7 (lock-only).

File-system scanner (FileSystemScanner)

  • Always includes roots Assets/Plugins and Assets/ThirdParty, plus configured scanFolders.
  • Each top-level subdirectory under each root becomes one sourceType = "folder" component.
  • Recursively scans Assets for native libraries with default extensions .dll, .so, .dylib, .bundle and configured extras.
  • Each native library becomes sourceType = "native-lib" and gets a SHA-256 hash.
  • Packages roots are honored only when showUnityPackages is enabled.

License file discovery (LicenseFileDetector)

  • Ranked filename categories: Primary (LICENSE, LICENCE, COPYING, EULA), Notice (NOTICE, COPYRIGHT, PATENTS), Attribution (AUTHORS, CONTRIBUTORS, ATTRIBUTION), and third-party notice bundle patterns.
  • Searches near component path/package root, legal subfolders, then walks parent folders until scope boundary.
  • For direct text population, uses the first Primary candidate only.
  • If maxTextFileBytesToRead is set, oversized files are skipped.

License scope boundaries (ComplianceScanner.ResolveLicenseScopeRoot)

  • Under Packages/: scope root is the package directory.
  • Under Library/PackageCache/: scope root is the package cache package directory.
  • Under Assets within a configured scan root: scope root is the first child directory below that root.
  • Under Assets outside configured roots: scope root is the first child under Assets.

Classification (LicenseClassifier)

  1. Skip manual-folder components.
  2. Skip components that already have a non-UNKNOWN license ID.
  3. If text contains SPDX-License-Identifier, prefer a known local catalog ID.
  4. Read package.json metadata (license, author, URLs).
  5. If still unresolved and text exists, run cosine similarity against normalized SPDX texts.
  6. Assign UNKNOWN if unresolved.

Similarity threshold uses licenseDetectionConfidenceThreshold. Third-party notice bundles are not matched as canonical single-license text.

Component keying and deduplication

  • Key base: idOrName|version|sourceType (lowercased).
  • Folder source appends primary path.
  • Native source appends hash if present, else path.
  • UPM-like sources (registry, git, local, embedded, upm, and transitive variants) do not append path/hash.
  • Dedup pipeline: merge by component key, dedup by name|primaryPath, then stable order via ComponentOrdering.Compare.

License Resolution Rules (LicenseResolver)

Resolution order

  1. Manual-folder special handling.
  2. Override entry from compliance-overrides.json by key (with legacy key fallback).
  3. Known catalog text by license ID.
  4. Detected file text fallback.
  5. UNKNOWN.

Manual-folder specifics

  • No ID and no text: UNKNOWN.
  • Known SPDX ID and no manual text: use catalog text.
  • Otherwise: use provided manual ID/text.

Override specifics

  • Override ID/text can replace detected values.
  • If override has ID but no text and the ID exists in catalog, catalog text is used.
  • URL fields are normalized and must pass absolute http/https validation.

Validation and Required Fields

Validation checks (ComponentValidation.ValidateComponent)

  • isUnknownLicense when license ID is UNKNOWN.
  • Missing required fields by policy: Author, Source URL, Changes Made.

License attribution policy highlights (LicenseAttributionPolicy)

  • Default/unknown: Author and Source URL required.
  • LicenseRef-Unity-Asset-Store-EULA: only Source URL required.
  • CC BY family (CC-BY-*, including NC/ND/SA): Author, Source URL, and Changes Made required.
  • Many SPDX licenses (MIT/BSD/Apache/LGPL/MPL/GPL/AGPL and others) make these fields optional.

URL validation rule (UrlUtility.ValidateUrl)

URLs must be absolute http:// or https://, well-formed, and include a host.

Overrides and Manual Entries

compliance-overrides.json stores three top-level arrays: entries, manualEntries, and packageImportRules.

Per-component override fields (OverrideEntry)

  • key
  • licenseId, licenseText, licenseTextSource
  • author, url, licenseUrl
  • creditLine/attribution, creditLineIsManual
  • changesWereMade, changesWereMadeSet
  • notes (supports IGNORE: marker prefix)
  • noticesEntry, noticesEntryIsManual
  • licenseYear, licenseHolder

Manual entry fields (ManualEntry)

  • folderPath (UI workflows require it to resolve under Assets)
  • displayName
  • author, url, licenseUrl
  • licenseId, licenseText
  • creditLine/attribution, creditLineIsManual, useAttributionText
  • changesWereMade, changesWereMadeSet
  • missing, includeWhenMissing
  • noticesEntry, noticesEntryIsManual

Manual entries map to runtime source type ManualFolder.

Ignore marker behavior

  • Mark Ignored... writes notes = "IGNORE: ...".
  • Ignored entries are hidden from "needs attention" counts in the editor list.
  • Ignore does not remove entries from build hook validation logic.
  • Ignore is not a hard exclusion from exported files.

Build Integration (BuildHook)

Pre-build sequence

  1. Scan, apply overrides, and resolve licenses.
  2. Filter Unity packages when showUnityPackages is false.
  3. Optionally export outputs when generateOnBuild is enabled.
  4. Run license and field policy checks.

Policy checks

  • Non-commercial licenses (IsCommerciallyRestricted): fail when failBuildOnNonCommercial, otherwise warn.
  • No-derivatives with confirmed modifications (IsNoDerivatives + changesWereMade = true): always fail.
  • No-derivatives without change confirmation (changesWereMadeSet unset): warn.
  • Strong copyleft (IsStrongCopyleft): warn.
  • Unknown/missing required fields: fail when failBuildOnUnknown, warn when warnOnUnknown.

Special case: No-derivatives components missing only Changes Made are warned separately and are not counted as fail conditions in the unknown/missing pass.

Output Format Details

compliance-manifest.json

Top-level fields in ComplianceManifest:

  • unityVersion
  • platform
  • timestampUtc
  • toolVersion
  • components[]

Each component includes identity/source, license, attribution, change metadata, and scanner confidence fields. ManifestWriter.WriteWithDiff computes diffs against prior manifest and preserves prior manifest content when no change is detected to avoid timestamp churn.

THIRD_PARTY_NOTICES.txt

  1. Title header
  2. "What Changed" section
  3. "Components" section with one block per component

Component blocks can include version, URLs, author, source/location, text source, credit line, license metadata, and either license text content or explicit UNKNOWN/incomplete placeholders. If noticesEntryIsManual is true, manual notices text is emitted verbatim.

CREDITS.md

  1. # Credits
  2. ## What Changed
  3. ## Component Credits

Auto line format: - <name> by <author> (<version>) - <url> with only available elements. Manual attribution replaces auto output when creditLineIsManual is true.

Package Import Monitor

PackageImportMonitor is enabled by default and tracks .unitypackage imports.

Prompt behavior for out-of-scope imports

  • No
  • Don't ask again
  • Move only
  • Yes, move & add license (requires valid source URL)

Move target and auto-license behavior

  • Move target: first configured scan root plus sanitized package folder name.
  • Move & add license creates/updates override with:
licenseId = LicenseRef-Unity-Asset-Store-EULA
licenseTextSource = Built-in profile
url = optional source URL from prompt

Determinism and Diff Stability

Deterministic behavior in implementation

  • Ordered component lists via ComponentOrdering.Order.
  • Stable ordering for diff lists (added, removed, changed).
  • Override store sorting before save: entries by key, manual entries by folder/display name, and package import rules by package/fingerprint.
  • Manifest no-change write optimization to avoid timestamp-only diffs.
