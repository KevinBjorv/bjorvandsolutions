Unity Serialization Migration Guard technical documentation

Unity Serialization Migration Guard supports controlled serialized field migration workflows through Audit, Coverage, Reserialize, Cleanup, and Settings operations.

It is centered on practical migration safety: identify risky renames, locate affected assets, and generate proof artifacts before changing the long-term mapping baseline.

1. Access paths

  • Unity menu: Tools / Refactor Migration Safety Suite / Open Window.
  • Unity menu: Tools / Refactor Migration Safety Suite / Open Project Settings.
  • Project settings path: Project / Refactor Migration Safety Suite.
  • Batch entrypoint: RefactorMigrationSafetySuite.Batch.RMSSBatchEntryPoint.Run.

2. RMSS window

Window title: RMSS.

Minimum size: 760 x 420.

Tab order:

  1. Audit
  2. Coverage Scan
  3. Reserialize
  4. Cleanup
  5. Settings

3. Audit tab

Controls

  • Git Diff Import: text input, diff file field, Browse Diff File, Preview Diff Import, Append Candidates, Replace Manual Mappings.
  • Manual Rename Mappings: add, clear, import JSON, export JSON, and remove row actions.
  • Per-row fields: Type Name, Old Name, New Name.
  • Run Audit.

Behavior and checks

  • Mappings are validated against indexed MonoScript assets in Assets.
  • Whitespace is normalized and duplicates are dropped.
  • Risk severity includes:
  • High: missing type, ambiguous type name, missing target field, non-serializable target field, missing FormerlySerializedAs.
  • Medium: token used by multiple resolved types, same legacy token still active on unresolved mapped type.
  • Low: case-only rename only.

Audit results clear when mappings change to avoid stale data. A remediation message is shown for missing FormerlySerializedAs.

4. Coverage tab

Controls

  • Run Coverage Scan.
  • Export Coverage JSON.
  • Export Coverage CSV.

Scope and behavior

  • Manual mappings must be valid: all Type, Old, New fields non-empty.
  • Requires Unity Force Text serialization.
  • Supported assets: .unity scenes, .prefab, standalone ScriptableObject .asset files, and parent assets with embedded ScriptableObjects.
  • Reports include candidate assets, scanned assets, unreadable assets, scope-complete state, affected assets, and token occurrence counts.
  • Per-field summary order: total occurrences, affected assets, type name, legacy field.

5. Reserialize tab

Controls

  • Scope dropdown: Coverage Results, Folder, Selection.
  • Folder object field for Folder scope.
  • Create Backup toggle.
  • Build Preview.
  • Run Reserialize.

Preview and execution

  • Preview includes source, asset count, skipped inputs, and warnings.
  • Run Reserialize is disabled until preview has at least one asset.
  • Scenes are reserialized through scene save flow, prefabs through prefab load/save, and other scriptable objects through AssetDatabase.ForceReserializeAssets.
  • Progress updates expose current stage, current path, and completed/total counters.

6. Cleanup tab

Controls

  • Evaluate Cleanup Proof.
  • Export Cleanup JSON.
  • Export Cleanup CSV.

Cleanup output

  • IsSafeToRemoveLegacyMappings bool.
  • IsSupportedScopeComplete bool.
  • RemainingLegacyTokenOccurrences.
  • Human-readable scope summary and notes.

Unsafe states include no coverage report, remaining legacy tokens, or incomplete scope.

7. Settings tab

Controls:

  • Enable Verbose Logging.
  • Create Backups By Default.
  • Default Export Directory.
  • Backup Directory.
  • Save Project Settings and Open Project Settings Page.

Settings are stored in ProjectSettings/RefactorMigrationSafetySuiteSettings.asset via ScriptableSingleton.

  • Default export directory: Logs.
  • Default backup directory: Logs.
  • Verbose logging and backup-by-default are enabled by default.

8. Project Settings provider

Provider path: Project / Refactor Migration Safety Suite.

Provider exposes the same fields as the Settings tab and supports explicit save persistence.

9. Rename mapping JSON schema

Schema version is 1.

Top-level fields: schemaVersion, mappings.

Each mapping includes:

  • typeName
  • oldName
  • newName
{
  "schemaVersion": 1,
  "mappings": [
    {
      "typeName": "MyCompany.CharacterState",
      "oldName": "oldHealth",
      "newName": "currentHealth"
    }
  ]
}

JSON is shared by Audit import/export and batch commands.

10. Batch mode

Command entry:

Unity -batchmode ... -executeMethod RefactorMigrationSafetySuite.Batch.RMSSBatchEntryPoint.Run

Supported commands:

  • audit
  • coverage
  • cleanup

Common options:

  • --mappingsJson <path> required for coverage and cleanup, optional for audit.
  • --jsonOut <path> optional output path.
  • --failOnRisk true|false for audit.
  • --failOnCoverage true|false for coverage/cleanup.

Exit codes:

  • 0 success.
  • 1 parse, input, or setup error.
  • 2 audit high-risk failure when fail-on-risk is enabled.
  • 3 coverage/cleanup failure when coverage failure is enabled.

11. Validation scripts

  • scripts/run-batch-audit.sh
  • scripts/run-batch-coverage.sh
  • scripts/run-batch-cleanup.sh
  • scripts/run-window-smoke-headful.sh

Scripts use a shared batch entrypoint and write to Logs.

12. Practical workflow

  1. Build or import mappings manually or from git diff.
  2. Export mappings for reuse and traceability.
  3. Run Run Audit and clear high and medium risks where possible.
  4. Enable Force Text and run Run Coverage Scan.
  5. Optionally use folder or selection scoped preview + reserialize.
  6. Run Evaluate Cleanup Proof.
  7. Export cleanup data before removing legacy fields or attributes.

13. Constraints and known hard limits

  • Coverage and cleanup depend on Force Text serialization.
  • Project Window selection scope only uses selection assets.
  • Reserialize is scope-limited and may skip unsupported assets.
  • Invalid backup folders under Assets or Packages are rejected and a project log fallback is used.
  • Unsupported diff hunks are skipped with warnings.
