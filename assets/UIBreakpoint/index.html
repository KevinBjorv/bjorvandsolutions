UI Breakpoint Lab Documentation

UI Breakpoint Lab is a Unity Editor utility for repeatable UGUI layout checks and visual regression captures.

It renders a selected scene Canvas or UI prefab against a saved matrix of viewport, orientation, safe-area, and text-expansion profiles, then presents screenshots and measured findings in one contact sheet.

Every isolated capture is labeled as an approximation. It is layout-analysis evidence, not a substitute for a player build on physical hardware.

Open UI Breakpoint Lab on the Unity Asset Store

1. Requirements and Compatibility

  • Minimum Unity version: 2022.3.41f1.
  • Release-tested editors: 2022.3.41f1, 6000.3.9f1, and 6000.4.4f1.
  • UGUI is required.
  • Built-in Render Pipeline and Universal Render Pipeline are the release-tested paths.
  • TextMesh Pro is optional. TMP overflow checks and expansion passes are enabled when its public assembly is available.

Other Scriptable Render Pipelines are conditional on accepting Unity's public standard camera render request. Unity versions older than 2022.3.41f1 are unsupported.

2. Installation

  1. Import the complete UI Breakpoint Lab package into the Unity project.
  2. Keep the supplied folders and .meta files together.
  3. Wait for compilation to finish and resolve the first Console error if Unity reports one.
  4. Open the tool from Tools > UI Breakpoint Lab.

If the menu item is missing, confirm the minimum Editor version, UGUI installation, complete import, and project compilation state.

3. Five-Minute Quick Start

  1. Open a scene containing a UGUI Canvas, or select a UI prefab.
  2. Open Tools > UI Breakpoint Lab.
  3. Choose the active scene, a Canvas, or the prefab as the target.
  4. Select Mobile Essentials.
  5. Keep the text passes you want to inspect and select Run Matrix.
  6. Filter the contact sheet to warnings or failures and open a finding.
  7. Save the configuration as a suite. After review, capture the completed run as a baseline.

Baseline capture accepts the current images as regression references. It does not mean every warning is resolved; rule severity and reasoned suppressions remain the suite's policy.

4. Targets and Profiles

Supported targets

  • Root Canvases in the active or selected scene.
  • One Canvas in an open scene.
  • One isolated UI prefab.

An unloaded scene can be opened additively long enough to clone its root Canvases, then closed without saving. Unity can still invoke project ExecuteAlways code during scene loading.

Profiles

Built-in profiles cover generic phone, tablet, desktop, ultrawide, and console-safe-area layouts. Custom profiles store pixel dimensions, DPI metadata, pixels per point, orientation, and a bottom-left-origin safe rectangle.

The source target is cloned into a preview scene. Source scenes and prefabs are not edited by analysis or text transformation.

5. Rules and Suppressions

Version 1 includes six versioned layout rules with measured evidence:

  • UBL001 Viewport Overflow.
  • UBL002 Parent Clipping.
  • UBL003 Safe-Area Violation.
  • UBL004 Text Overflow.
  • UBL005 Overlap heuristic.
  • UBL006 Touch-target checks.

Suites can override severity per rule. Suppressions require a reason and can target an object, hierarchy, profile and suite combination, or the whole suite. Use the narrowest scope that expresses the intended design.

Suppressed findings retain their evidence and suppression state in reports.

6. Text Expansion and Pseudolocalization

Available passes are Original, 120%, 150%, 200%, and deterministic ASCII pseudolocalization.

  • Transformations run only on preview clones.
  • Rich-text tags and composite-format placeholders are preserved.
  • Static UnityEngine.UI.Text is supported directly.
  • Public TMP text components are supported when TMP is available.
  • Custom text sources can use the shipped preview-text adapter.

Unity Localization package integration and platform-font equivalence are not included in version 1.

7. Baselines and Visual Diffs

A baseline is an explicitly accepted set of PNG captures and versioned metadata. Baseline identity prevents unrelated profile, text, tool, or rendering environments from being treated as interchangeable.

  • Compare side by side, with a wipe, or through a heatmap.
  • Configure integer RGBA tolerance from 0 through 255.
  • Use small ignored regions only for understood, intentionally variable content.
  • Update one accepted profile and text-pass entry, or separately confirm a complete baseline replacement.

Pixel-dimension and color-space mismatches block numeric comparison. Unity-version, render-pipeline, and graphics-API differences produce prominent compatibility warnings.

8. Reports and Annotations

In-Editor reports are versioned ScriptableObject assets containing captures, findings, environment metadata, annotations, progress outcome, and approximation warnings.

Reports can be exported as JSON schema version 1 or portable HTML. Keep the HTML file beside its complete sibling-assets folder so screenshot references remain valid.

Each finding records its rule and version, evidence type, effective severity, measured values, top-left-origin screenshot coordinates, stable source identity, and suppression state.

9. Command Line and CI

The command-line runner executes the same suite, profiles, rules, severity policy, suppressions, baselines, diff algorithm, and exporters as the Editor window.

Unity.exe -batchmode -projectPath "C:\work\MyGame" ^
  -executeMethod KevinBjorvand.UIBreakpointLab.UIBreakpointCli.Run ^
  -ublSuite "Assets/UI Tests/Main Menu.asset" ^
  -ublOutput "C:\work\artifacts\ui-breakpoint-lab" ^
  -logFile "C:\work\artifacts\ui-breakpoint-lab.log"

Screenshot capture requires graphics. Do not use -nographics. The suite target must be a saved scene, persistent Canvas reference, or prefab; dynamic Active Scene targets are rejected in batch mode.

Exit codes

  • 0: complete matrix, verified source integrity, configured gate passed.
  • 2: configured finding, required comparison, or diff-budget failure.
  • 3: invalid arguments, suite path, suite validation, or target.
  • 4: the run or export could not produce trustworthy complete evidence.

Add -ublFailOnWarnings when configured warnings should also fail the gate. Archive the complete output directory and Editor log.

10. Accuracy and Version 1 Limits

UI Breakpoint Lab uses documented public Unity APIs and does not reflect into Game view or Device Simulator internals.

Public Editor APIs cannot replace Screen.width, Screen.height, Screen.safeArea, Screen.dpi, runtime orientation, or device services. Preview-aware project code can use the shipped preview context and safe-area adapter.

Version 1 does not include UI Toolkit validation, XR or custom render-texture UI, physical-device rendering, performance simulation, exact platform-font rendering, automatic gameplay navigation, automatic layout rewriting, or Unity Localization package integration.

11. Troubleshooting and Support

Blank or missing CI screenshots

Remove -nographics, confirm the agent has a usable graphics environment, and confirm the active pipeline accepts the public render request.

Repeated captures differ

Check the Unity version, pipeline, color space, graphics API, fonts, animation, generated assets, and time-, random-, network-, or device-dependent preview content.

Project code ignores the simulated safe area

Code that reads Screen.safeArea directly receives the Editor value. Use the preview context or safe-area adapter for opt-in preview behavior.

Source scene or prefab becomes dirty

Do not save the unexpected changes. Cancel the run, preserve the target, phase, Editor log, and version-control diff, and report it as a defect.

Contact support, review the license, or browse the UI Breakpoint Lab articles.
