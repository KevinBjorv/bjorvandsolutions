Unity Editor Tool

Detect and prevent build size regressions before they slow down your release.

Build Size Guard captures snapshots after builds, diffs snapshots, enforces budgets, and generates CI-friendly report artifacts with deterministic exit codes.

The problem

Build size can jump unexpectedly between builds. Teams then have to answer the same release-critical questions under time pressure: what changed, what contributed most to the change, whether budgets were violated, and whether CI should fail the build.

What Build Size Guard does

  1. Capture snapshot data after each build from BuildReport
  2. Diff baseline and current snapshots
  3. Evaluate budgets and produce violations
  4. Output concise summaries plus JSON/Markdown report artifacts
Snapshots store a ranked top-N contributor list plus grouped remainder totals. This keeps snapshots small and diffs fast.

Why this approach

Automatic capture

Snapshots are captured after builds with optional retention per build target.

Focused diffs

Top contributors plus grouped remainder totals keep comparisons practical and reviewable.

Editor-only runtime impact

The tool runs in Unity Editor and batchmode editor and does not ship in your player build.

What you get (generated artifacts)

Snapshots (`*.json`)

Stored under the configured data root (default `BuildSizeGuardData/snapshots/`) with build identifiers, size metrics, and contributor/grouped totals.

Reports (`*.json`, `*.md`)

Optional report outputs for CI artifacts and review workflows (default `BuildSizeGuardData/reports/`).

Latest passing index

Baseline index file (`BuildSizeGuardData/latest_passing_snapshots.json`) used for `latestPassing` resolution in CI.

How it works in practice

Diff data included

  • Total delta and percent delta
  • Increases, decreases, new, removed, and moved items (when GUID exists)
  • Grouped totals by category and folder, including remainder grouping

Addressables rollups (when available)

  • Grouped totals by group
  • Grouped totals by bundle
  • Grouped totals by asset

Addressables totals are captured when `buildlayout.json` is present and readable.

Budget policies supported

  • Report total (`BuildReport.summary.totalSize`)
  • Report delta and report percent increase
  • Output artifact total, delta, and percent increase
  • Per-category budgets (texture, audio, mesh, shader, code, other)

CI / batchmode runner

Batchmode execute method:

BuildSizeGuard.Editor.Cli.BuildSizeGuardBatchRunner.Run

CLI supports baseline/current selection, match mode, optional report paths/prefix, and violation fail behavior.

Exit codes:

  • `0`: success
  • `2`: budget/regression violation (with failure enabled)
  • `3`-`7`: argument, baseline, load, write, or runtime error states

FAQ

Q: Why do report total size and output artifact size differ?

A: They are related but not identical metrics, and platform packaging can vary.

Q: Why is contributor detail limited?

A: Snapshot detail depends on what Unity exposes and your `Top Contributor Limit` setting. Higher limits store more detail.

Q: What does `latestPassing` need in CI?

A: Persist the data root between runs (cache or artifact restore), so the next run can resolve older snapshots.

Q: What if Addressables data is missing?

A: If `buildlayout.json` is missing, malformed, or stale, Addressables totals are empty for that snapshot.

Q: Does this ship into my player build?

A: No. Build Size Guard is Editor-only.
