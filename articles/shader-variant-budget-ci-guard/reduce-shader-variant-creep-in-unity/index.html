How to Reduce Shader Variant Creep in Unity

Use snapshot-based visibility to catch shader variant growth before it turns into a release problem.

March 11, 2026 • 8 min read • Kevin Ahlgren Bjorvand

The reliable way to reduce shader variant creep in Unity is to capture variant snapshots consistently, compare them against a baseline, and review the top offenders before variant growth becomes accepted as normal.

Updated March 11, 2026 Tool homepage, docs, support, and license linked in the sidebar

Why does shader variant creep stay hidden for so long?

Shader variant growth is usually gradual. A new keyword path, an extra platform combination, or a build profile change can increase counts without producing a dramatic failure right away.

If the only source of truth is the raw Editor.log, teams are left scrolling through text instead of working with a stable baseline and a meaningful diff.

  • The growth is often spread across many normal-looking changes.
  • Raw logs are not a good review surface.
  • Mixed targets and profiles make ad hoc comparison even harder.

What does Shader Variant Budget & CI Guard actually capture?

The tool reads Editor.log, extracts summary totals and per-shader contributor data, stores deterministic snapshot JSON, and carries parser diagnostics alongside the numbers. That matters because you need both the totals and the confidence that the parser saw a valid snapshot.

The same pipeline runs in the editor and in batchmode, so the local comparison path and the CI path stay aligned.

  • Total variant counts.
  • Per-shader contributor totals.
  • Snapshot metadata such as target and build profile when available.
  • Parser diagnostics and completeness information.

Make shader variant growth visible before it becomes background noise.

Shader Variant Budget & CI Guard turns Editor.log output into deterministic snapshots, compare results, and budget findings that teams can act on quickly.

How do you use snapshots to control the creep instead of reacting late?

The fastest routine is to capture a snapshot after a build, refresh the browser list, and compare the new snapshot against the nearest valid baseline. In the editor window, Compare Selection uses a deterministic baseline order that prefers the same target and build profile when possible.

That baseline logic matters because it reduces false comparison pairs. You get a result that is more likely to explain real growth instead of noise from unrelated targets.

  1. Capture snapshots consistently after the builds you care about.
  2. Compare against the closest valid baseline for the same target or build profile when possible.
  3. Review top contributors and diff results before the growth is normalized.
  4. Promote clean runs to a latestPassing baseline when the project uses CI.

What is the tool not trying to be?

Shader Variant Budget & CI Guard is editor-only. It is not a runtime package and it does not add in-game diagnostics or runtime optimization logic.

The current editor window also does not try to be a full report exporter. Diagnostics export is available in the window, but full JSON and Markdown compare reports are batchmode features.

  • No runtime package or in-game UI.
  • No snapshot delete button in the current editor window.
  • No full compare report export from the editor window itself.
